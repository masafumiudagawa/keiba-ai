買い目最適化 - ロジック詳細 & 機能拡張設計

現在のロジック解説 + 他ツール参考の追加機能設計 | 2026-06-09

Part A: 現在のロジック

A1. JRA控除率（馬券の原価）

JRAは馬券売上から一定割合を差し引いてから配当を計算する。この控除率がベースライン。

馬券種別控除率還元率意味
単勝 / 複勝20%80%期待値0.80が損益分岐点
馬連 / ワイド22.5%77.5%
三連複25%75%
三連単27.5%72.5%控除率最大だが配当も最大
馬券を買う時点でJRAに20-27.5%取られている。期待値1.0を超えるには、自分の予測が市場（=他の買い手全体）より正確でなければならない。

A2. オッズ推定（実オッズ未取得時）

単勝オッズ = (還元率 / AI予測確率) × 市場歪み補正 市場歪み補正: 人気馬(確率>15%): ×1.1 ← 実オッズは予測より高くなりがち 中位(8-15%): ×1.0 人気薄(<8%): ×0.9 ← ロングショットバイアス（過大評価されがち） 例: クロワデュノール 予測17.7% → 0.80 / 0.177 × 1.1 = 4.97倍（推定） → 実際のオッズが5.0倍以上ならプラス期待値
現在の問題: 実オッズなしだと推定オッズ = 還元率/確率 なので、期待値は常に0.8-0.9（マイナス）。木曜以降に実オッズを入力すると期待値1.0超えの馬券が特定できる。

A3. 的中確率の計算

馬連（A-Bが1-2着、順不同）

P(馬連 A-B) = P(A1着)×P(B2着|A1着) + P(B1着)×P(A2着|B1着) P(B2着|A1着) = P(B勝率) / (1 - P(A勝率)) ← 条件付確率 例: クロワ(17.7%) - ミュージアム(11.8%) = 0.177 × (0.118/0.823) + 0.118 × (0.177/0.882) = 0.177 × 0.143 + 0.118 × 0.201 = 0.025 + 0.024 = 4.9%

ワイド（A-B両方3着以内）

P(ワイド A-B) = P(A複勝) × P(B複勝|A複勝) × 補正 補正 = (頭数 / (頭数-1)) × 0.85 ← 条件付確率の近似 例: クロワ(複勝45%) - ミュージアム(複勝38%) = 0.45 × 0.38 × (18/17) × 0.85 = 0.171 × 1.059 × 0.85 = 15.4%

三連複（A-B-C全員3着以内）

P(三連複 A-B-C) = P(A複) × P(B複) × P(C複) × 組合せ補正 組合せ補正 = (3/N) × (2/(N-1)) × (1/(N-2)) × N³/6 ← 3つの枠に3頭を配置する条件付確率 例: クロワ-ミュージアム-メイショウ = 0.45 × 0.38 × 0.37 × 補正 ≈ 8-12%

A4. リスクレベル別の買い方戦略

低（堅実）中（バランス）高（大穴狙い）
対象馬上位3頭のみ上位5頭上位7頭（穴馬含む）
点数5点（厚く少なく）8点12点（薄く広く）
優先種別単勝・複勝・ワイド全種均等馬連・三連複
ソート基準的中率が高い順期待値順オッズが高い順
配分本命に50%集中上位に30%傾斜ほぼ均等

A5. 金額配分ロジック

各買い目の金額 = 予算 × (重みi / 重みの合計) 重みi = 1.0 + top_heavy × (N - i) / N top_heavy: low=0.5, medium=0.3, high=0.1 例: 予算10,000円, 5点, low (top_heavy=0.5) 重み = [1.5, 1.4, 1.3, 1.2, 1.1] → 合計6.5 金額 = [2,300, 2,200, 2,000, 1,800, 1,700] ← 100円単位に丸め

Part B: 現在の問題点

問題1: 実オッズなしでは期待値が常にマイナス
推定オッズ = 還元率/確率 なので、EV = 確率 × オッズ = 還元率(0.75-0.80) < 1.0。
→ オッズ取得後でないと「プラス期待値の買い目」を特定できない。
問題2: 買い方の自由度が低い
現在は「AI任せの自動最適化」のみ。ユーザーが軸馬を指定したり、フォーメーションを組んだりできない。
問題3: フォーメーション / 流し / ボックスに非対応
実際の馬券購入ではフォーメーションが最も効率的な買い方だが、現在は組合せ列挙のみ。

Part C: 追加機能設計

C1. 軸馬指定モード NEW

ユーザーが「この馬は絶対来る」と判断した馬を軸に固定して買い目を生成。

UI設計

  • 予測一覧テーブルの各行に「軸に設定」ボタン
  • 軸馬（1-2頭）を選択 → 相手馬はAIが自動選出
  • 1軸流し / 2軸流し の切替

ロジック

■ 1軸流し（軸A → 相手B,C,D...） 馬連: A-B, A-C, A-D, ... 三連複: A-B-C, A-B-D, A-C-D, ... ■ 2軸流し（軸A,B → 相手C,D,E...） 三連複: A-B-C, A-B-D, A-B-E, ...

C2. フォーメーション組立 NEW

三連複・三連単で「1着候補」「2着候補」「3着候補」を個別に指定。

■ 三連複フォーメーション 1着候補: クロワ, ミュージアム 2着候補: クロワ, ミュージアム, メイショウ, レガレイラ 3着候補: クロワ, ミュージアム, メイショウ, レガレイラ, ダノン, タガノ → 自動で点数計算 & 重複排除 → 合計 XX 点 × 100円 = XX,XX0円 ■ UI: 3つのエリアに馬をドラッグ&ドロップ

C3. ボックス買い NEW

■ 三連複BOX: 選択馬から全組合せ 5頭BOX → C(5,3) = 10点 6頭BOX → C(6,3) = 20点 ■ 馬連BOX: 4頭BOX → C(4,2) = 6点 ■ UI: チェックボックスで馬を選択 → 点数を即時計算

C4. 合成オッズ計算 NEW

■ 合成オッズ = 総投資額 / 最低払戻額 例: 馬連3点買い A-B 2,000円 × 8.5倍 = 17,000円 A-C 1,500円 × 12.0倍 = 18,000円 A-D 1,000円 × 25.0倍 = 25,000円 総投資: 4,500円 最低払戻: 17,000円 合成オッズ: 17,000 / 4,500 = 3.78倍 ■ 「どれが当たっても最低XX倍」を保証する均等投資モードも

C5. オッズ手入力 & リアルタイム期待値 NEW

木曜以降にオッズが確定したら、ユーザーが手入力（またはスクレイピング自動取得）。

■ 期待値 = AI予測確率 × 実オッズ クロワ単勝: AI 17.7% × 実オッズ 2.5倍 = EV 0.44 ← 割に合わない クロワ単勝: AI 17.7% × 実オッズ 7.0倍 = EV 1.24 ← プラス期待値！ ■ EV > 1.0 の買い目をハイライト（= AIが市場より高く評価している） ■ これが本来の「AI予測の価値」= 市場との乖離を見つける
これが最重要機能: AIの真価は「市場が過小評価している馬」を見つけること。実オッズとの比較で初めてプラス期待値の買い目が特定できる。

C6. 収支シミュレーション NEW

■ 「もし1着がXX, 2着がYY, 3着がZZだったら」のシミュレーション 入力: 買い目リスト + 着順予想（ドロップダウン） 出力: 的中馬券: 馬連 A-B → 2,000円 × 8.5倍 = 17,000円 三連複 A-B-C → 1,500円 × 45.0倍 = 67,500円 合計回収: 84,500円 投資: 10,000円 収支: +74,500円 (845%)

Part D: 実装ロードマップ

1
オッズ手入力 & 期待値表示 最優先
木曜にオッズ確定後すぐ使える。EV>1.0の買い目をハイライト。
2
軸馬指定 + 流し買い
「クロワ軸で相手5頭」のような買い方。最も一般的。
3
フォーメーション & ボックス
点数計算 + 合計金額の即時表示。
4
合成オッズ計算
複数買い目の最低保証倍率を表示。
5
収支シミュレーション
着順を仮定して払い戻しをシミュレート。
