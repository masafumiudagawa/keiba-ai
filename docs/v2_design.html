1. 改修の全体像

改修A: スコアリングウェイト調整UI

23カテゴリのウェイトをスライダーで調整 → 予測結果がリアルタイムで変化。「世論の影響を下げたい」「上がり3Fをもっと重視したい」が画面上で即反映。

改修B: 汎用レース対応

宝塚記念専用 → 任意のレースに対応。レースごとの距離・コース・傾向パラメータを設定可能。JRA全G1・重賞に展開。

3-3. レース追加フロー

  1. レース基本情報入力: 名前・日付・競馬場・距離・芝/ダート・グレード
  2. 出走馬入力: CSVアップロード or 手動入力 or netkeibaから自動取得（レースID入力）
  3. 傾向パラメータ: 過去データから自動推定 or ユーザー手動設定
    • 馬齢バイアス: 何歳が有利か
    • 脚質バイアス: 前有利 or 差し有利
    • 距離適性の重要度
    • 重馬場の影響度
  4. データ収集: 出走馬の戦績・騎手・調教を自動収集
  5. 予測実行: 汎用スコアリング + レース固有バイアスで計算

6. ウェイト調整のフロントエンド実装詳細

useScoring.ts（カスタムHook）

// 全馬の特徴量をAPIから1回取得 const features = await fetch('/api/races/takarazuka_2026/features') // スライダー変更時にJSでスコア再計算（サーバー不要） function recalculate(features, weights) { return features.map(horse => { let score = 0 score += horse.age_score * weights.age score += horse.recent_form_score * weights.recent_form score += horse.g1_score * weights.g1_record score += horse.last3f_score * weights.last_3f score += horse.speed_score * weights.speed_figure score += horse.yt_news_score * weights.public_opinion // ... 23カテゴリ分 return { ...horse, total_score: score } }) // → 正規化して確率に変換 }

スライダーを動かすたびに recalculate() が走り、テーブルが即座に更新される。
