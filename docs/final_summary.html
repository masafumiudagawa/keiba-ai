1. システム概要

予測エンジン
ルールベース 23カテゴリスコアリング。各カテゴリにユーザーがウェイト（0.0〜2.0x）を設定し、フロントエンドでリアルタイムに順位・確率を再計算。
買い目最適化
バリュー比率ベース（v3）。AI確率と実オッズ（市場評価）を比較し、AIが市場より高く評価している馬・組み合わせのみを推奨。
技術スタック
Backend: FastAPI (Python 3.13) | Frontend: React 19 + TypeScript 6 + Tailwind 4 + Vite 8 | Deploy: Render

3. 予測エンジン（23カテゴリ × 4Tier構造）

設計思想: 23の独立したスコアカテゴリで各馬を評価。ML研究・プロ予想家の知見・学術論文に基づき、4段階のTier（重要度階層）で配点を設計。 各カテゴリのmaxスコアは5〜32の範囲に収め、1つのカテゴリが予測全体を支配しない構造。UIのスライダーは17グループに集約して操作性を確保。

3-A. Tier1: 最重要カテゴリ（max 25-32）

ML研究で一貫して予測力が高いファクター。合計maxの約40%を占める。

#カテゴリkey計算式max根拠
1前走着順last_finish(18 - 前走着順) × 1.525.5ML研究で常に最上位の予測因子
2スピード指数speed_figure(スピード指数 - 90) × 1.230能力の絶対値。芝での精度低下を考慮し係数抑制
3上がり3Flast_3f(37 - ベスト上がり3F) × 5
※32〜37秒でクリップ		25中距離G1で最速馬の複勝率が極めて高い
4G1勝利g1_winsmin(ln(1 + G1勝数) × 15, 25)25最高峰での実績。対数で差を圧縮
5距離適性distance_aptitude(距離±400m好走率×20 + (10-平均着順)×1.5 + 経験値) × 0.728G1馬はすでに適性ありのため係数0.7で抑制

3-B. Tier2: 重要カテゴリ（max 10-25）

予測に有意に寄与するが、Tier1ほど決定的ではないファクター。

#カテゴリkey計算式max根拠
6馬齢ageconfig.trends.age_bias[年齢]
※レースごとに設定（宝塚:5歳=25, ダービー:3歳=25等）		25学術研究ピーク4.45歳。config.jsonで柔軟に設定
7騎手G1実績jockey_g1ln(1 + 騎手G1勝数) × 4.520ML分析で常に上位。例: ルメール48勝→17.5pt
8G1複勝g1_placesmin(G1複勝回数 × 3, 18)18安定感の指標。G1勝利(max25)を超えない設計
9通算勝率career_win_rate通算勝率 × 1515安定した予測因子。70%勝率なら10.5pt
10騎手勝率jockey_win_ratemin(シーズン勝率 × 50, 15)15現在の好調さ。例: 20%→10pt, 10%→5pt
11調教training(調教強度 - 3) × 6.5
※1-5段階		13追い切り速い馬は遅い馬の3-10倍の勝率
122走前着順second_last_finish(18 - 2走前着順) × 0.712安定感の指標。前走より減衰（1.5→0.7）

3-C. Tier3: 中程度のカテゴリ（max 5-12）

レース条件や相手関係に依存して重要度が変わるファクター。

#カテゴリkey計算式max根拠
13対戦成績head_to_headmin(対戦勝率 × 12, 12)
※対戦データあり時のみ		12直接対決の相性。サンプル少のため抑制
14厩舎力trainer_scoreln(1 + 厩舎G1勝数) × 4.211学術研究で「予測に有意な変数」
15コース経験venue_experiencemin((同競馬場好走率×8 + min(出走数,4)×1.0) × 0.8, 10)10競馬場特性の慣れ。初コースの馬は0pt
16脚質running_styleconfig.style_bias[脚質] × 1.2
※宝塚: 逃げ=8, 先行=5, 差し=2, 追込=0		9.6展開依存。config.jsonで柔軟に設定

19体重推移weight_trendstable=7, increasing=4, decreasing=27±2kg安定が最高勝率（JRAデータ）

3-E. 配点設計の根拠

設計原則詳細
maxレンジの均等化旧: 1.5〜40（27倍差） → 新: 5〜32（6倍差）。1カテゴリが予測を支配しない
ML研究との整合LightGBM/XGBoostの feature importance 上位（前走, SP指数, 騎手）をTier1に配置
相関カテゴリの抑制pedigree_distance×venue r=0.98, youtube×news r=0.97 → Tier4に抑制し二重計上を防止
学術研究の反映年齢ピーク4.45歳, 休養明け好走率向上, 馬体重±2kg最高勝率 等を配点に反映
世論の位置づけ世論は「市場の集合知」として一定の価値あり。キャップ(max12)内で差別化を確保しつつ、Tier1を超えない設計

3-F. スコア → 確率 変換フロー

1. 各馬の合計スコア = Σ(23カテゴリスコア × UIグループウェイト) ※ UIは17グループ。1スライダーが複数のscoreキーに適用される ※ 例: 「近走」スライダー → last_finish + second_last_finish に同じウェイト適用 2. 正規化: prob = (合計 - 最小値) / (最大値 - 最小値) ※ 全馬同スコアの場合は等確率(1/N)にフォールバック 3. 確率化: prob = prob / Σprob （合計100%に正規化） 4. ソート → 順位 → 印（◎○▲△△）

5. 買い目最適化エンジン v3（バリュー比率ベース）

v4.0で全面書き換え。 旧v3.0のKelly基準+推定オッズ方式を廃止。実オッズ（odds.csv）とAI確率を比較する「バリュー比率」ベースに変更。

5-A. コアコンセプト

バリュー比率 = AI確率 / 市場確率 = AI確率 × 実オッズ 例（東京ダービー 2026）: フィンガー: AI=23.8%, オッズ=4.3x → 市場23.3% → 比率1.02x → 適正 ロウリュ: AI=10.0%, オッズ=133.6x → 市場0.7% → 比率13.3x → 狙い目 シルバーレシオ: AI=12.1%, オッズ=1.1x → 市場90.9% → 比率0.13x → 過大評価

5-B. 判定ロジック

判定条件意味
狙い目 (STRONG_BUY)バリュー比 ≥ 1.5 かつ AI確率 ≥ 5%AIが市場より大幅に高評価。確信度も十分。
妙味あり (BUY)バリュー比 ≥ 1.2 かつ AI確率 ≥ 3%AIが市場よりやや高評価。
適正 (FAIR)0.8 ≤ バリュー比 ≤ 1.2AIと市場がほぼ一致。
過大評価 (OVERVALUED)バリュー比 < 0.8 / AI確率低すぎ市場が過大評価。消し。
999倍の超不人気馬はAI確率も低い（1-2%）ため、AI確率の閾値フィルタにより買い目候補から自動除外される。「オッズが高い＝期待値が高い」にはならない。

5-C. 6つの最適化モード

モード内容
バリュー分析全馬のAI順位 vs 人気順位を比較表示。バリュー比率順にソート。
AI最適化バリュー馬のみで買い目生成 → ケリー基準で金額配分
3パターン比較堅実/バランス/大穴の3パターンを横並び比較
軸馬流し1-2頭を軸に指定 → 相手をAIが選出
フォーメーション1着/2着/3着候補を個別選択 → 点数+期待値計算
ボックスN頭BOX → 全組合せ+期待値計算
